FreeRadius - Tooling OpenSSL

Installing a working copy of FreeRadius (not from Ubuntu).

The first thing to do is to check what version is installed

1

openssl version

In my case “OpenSSL 1.1.1 ” was the result.

OpenSSL 1.1.1j

Download openssl-1.1.1j from here and save it into ~/Downloads directory:

1
2

cd ~/Downloads
wget https://www.openssl.org/source/openssl-1.1.1j.tar.gz

We will install the new version at /opt/openssl. To do that we need to create and change directory

1
2

sudo mkdir /opt/openssl
cd /opt/openssl

Extract the downloaded zip file into this directory:

1
2

sudo tar xfzv ~/Downloads/openssl-1.1.1g.tar.gz --directory /opt/openssl
cd openssl-1.1.1j

Execute the following command to create the Makefile:

1

sudo ./config --prefix=/opt/openssl --openssldir=/opt/openssl/ssl

Where /opt/openssl is the home directory and /opt/openssl/ssl is the directory where OpenSSL will store certificates and private keys.

Execute the Makefile (this will take some time)

1
2

sudo make 
sudo make install

OpenSSL new version 1.1.1g is installed.

Multiple Instances

Right now there are two installations: the original and this one.

I won’t delete the original, but only set the new one as the default. Backup the original

1

sudo mv /usr/bin/openssl /usr/bin/openssl.old

In the literature there are references to applications that expect openssl to be at the original directory. To maintain compatibility, and avoiding the need to alter the environment variable PATH, we will create a symbolic link /usr/bin/openssl pointing to /opt/openssl/bin/openssl

1
2

sudo ln -s /opt/openssl/bin/openssl /usr/bin/openssl
ls -lisah /usr/bin/openssl

Setting the Module

We need to modify the module loader configuration in /etc/ld.so.conf.d/openssl.conf, open the file and edit it using vi /etc/ld.so.conf.d/openssl.conf so that it reads as follows

1

/opt/openssl/lib

Now, reload the modules

1

sudo ldconfig

And finally, we can verify that everything is correct

1
2
3

which openssl
openssl version
openssl

Clean up

Reboot your system to make things permanent and execute the last three commands again, targeting, obviously, the same outcome.

By now you have OpenSSL new version installed and working correctly. But if you try to download any of the previous files, for instance openssl-1.1.1g.tar.gz, you will get the following error:

This error is to be expected. The Certificate Authority “Let’s Encrypt Authority X3” that issued the server certificate is not in OpenSSL certificate and private key directory (/opt/openssl/ssl). If this is the desired behavior skip what follows and you have OpenSSL 1.1.1g completely installed.

If this is not your desired behavior, you have two options

  1. Copy all certificates in /etc/ssl/certs/ to /opt/openssl/ssl/certs
  2. Make /opt/openssl/ssl/certs a symbolic link pointing /etc/ssl/certs/ files (preferred).

To Implement the preferred approach, issue the following command

1

sudo ln -s /etc/ssl/certs/. /opt/openssl/ssl/certs/
